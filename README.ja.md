MediaLIB は、ローカルディスク、外付けドライブ、NAS、SMB/FTP のネットワークフォルダ、そして Emby、Jellyfin、Plex に散らばった映画、ドラマ、アニメ、ドキュメンタリー、バラエティ、音楽をひとつにまとめます。ひとつの画面から、閲覧、検索、お気に入り、続きを見る、再生まで自然に扱えます。

フォルダを勝手に管理したり、メディアファイルを移動・変更したりすることはありません。MediaLIB は、きれいに整った「メディアの索引センター」のようなものです。ファイルは元の場所にそのまま残り、分類、カバー画像、再生履歴、お気に入り、プレイリスト、オフラインキャッシュ、メタデータの修正は App のローカルデータとして保存されます。

</div>

## こんな人に向いています

- 動画や音楽が Mac、外付けドライブ、NAS、ホームサーバーに分かれていて、普段は Finder でフォルダをたどって探している。
- 映画、ドラマ、アニメ、音楽をひとつの App でまとめて見たい。
- Emby、Jellyfin、Plex を使っているけれど、ローカルファイルもたくさん持っている。
- シンプルで静か、macOS らしい見た目のメディアライブラリが好き。
- プライベートなフォルダをロックし、パス、ファイル名、再生の痕跡を見えないようにしたい。

## 主な機能

### メディアをひとつの場所で管理

MediaLIB は、ローカルフォルダ、ネットワークデバイス、Emby、Jellyfin、Plex に対応しています。メディアソースを追加するときは、接続元を選び、接続を完了し、分類とスキャン方法を確認するだけです。

<div align="center">
  <img src="./assets/add-source.png" alt="メディアソースを追加" width="78%" />
</div>

ローカルフォルダやマウントされたネットワークフォルダは、映画、ドラマ、アニメ、ドキュメンタリー、バラエティ、音楽、その他、Vault に分類できます。迷った場合は自動識別も使えます。リモートメディアサーバーは独立した入口として表示され、ローカルの動画や音楽と直接混ざることはありません。

### 映画、ドラマ、アニメ

MediaLIB はフォルダを再帰的にスキャンし、一般的な動画ファイルを認識します。さらに、ファイル名やフォルダ構成から内容の種類をできるだけ判断します。エピソードはシリーズごとにまとめられ、シーズンと話数の順に並びます。`S01E01`、`第01季 第02集`、`EP01` などのよくある命名にも対応しています。

<div align="center">
  <img src="./assets/movie-library.png" alt="映像ライブラリ" width="86%" />
</div>

作品には、お気に入り、見たい、視聴済み、未視聴、視聴中といったマークを付けられます。1〜5 星の個人評価も追加できます。MediaLIB では、タイトル、年、スコア、個人評価、最近追加、再生回数などでライブラリを見られます。

自分で整理したい作品には手動コレクションを作れます。App に自動で絞り込んでほしい場合は、種類、視聴状態、年、スコア、ソースなどの条件からスマートコレクションを作れます。よく使うコレクションはホームに表示することもできます。

<div align="center">
  <img src="./assets/series-detail.png" alt="シリーズ詳細" width="86%" />
</div>

### 音楽ライブラリとプレイリスト

音楽は動画と並ぶ、独立したメディアライブラリとして扱われます。MediaLIB は音声タグを読み取り、タイトル、アーティスト、アルバム、トラック番号、年、長さ、埋め込みカバー、歌詞をできるだけ認識します。

<div align="center">
  <img src="./assets/music-library.png" alt="音楽ライブラリ" width="86%" />
</div>

曲、アルバム、アーティスト、プレイリスト、最近再生した項目から閲覧できます。お気に入りにした曲は、上部に固定されたお気に入りプレイリストに入ります。自分のプレイリストを作ったり、現在の再生キューをプレイリストとして保存したりすることもできます。プレイリストは MediaLIB 内部の索引と順序だけを記録し、音楽ファイルを移動、コピー、リネームすることはありません。

### もっと没入感のある音楽再生

音楽プレイヤーには、下部のミニプレイヤーと展開ページの 2 つの状態があります。曲をダブルクリックすると、まずミニプレイヤーが表示されます。曲情報をクリックすると、フルサイズの再生画面に展開します。展開ページはアルバムカバーから色を取り、カバー、操作バー、キュー、歌詞カードをまとめて表示します。

<div align="center">
  <img src="./assets/music-player.png" alt="音楽プレイヤー" width="78%" />
</div>

歌詞は、埋め込み歌詞または同名の `.lrc` / `.txt` ファイルを優先して読み込みます。タイムスタンプ付きの LRC は自動スクロールでき、拡張 LRC では単語ごと、または区切りごとのハイライトにも対応します。単語単位の正確なタイミングがない場合でも、MediaLIB は歌詞のリズムから行内の進み具合を推定し、より自然に音楽へ追従させます。

### 内蔵ビデオプレイヤー

動画は内蔵プレイヤーで再生できます。必要に応じてシステムプレイヤーを使うこともできます。内蔵プレイヤーは、一般的な動画コンテナ、字幕と音声トラックの切り替え、外部字幕、再生速度、音量、フルスクリーン、最前面表示、続きから再生、スクリーンショット、チャプター、ブックマーク、A-B リピート、オープニングやエンディングのスキップなどに対応しています。

<div align="center">
  <img src="./assets/video-player.png" alt="ビデオプレイヤー" width="86%" />
</div>

リモート動画では、画質選択とローカルキャッシュにも対応しています。ネットワークが不安定なときに先に準備したり、一時的にオフラインで見たいときに便利です。

### Vault

Vault はプライベートなコンテンツ向けの場所です。初回利用時に 4〜8 桁の数字パスコードを設定でき、その後は Touch ID またはパスコードでロック解除できます。

<div align="center">
  <img src="./assets/vault.png" alt="Vault" width="86%" />
</div>

ロック中、MediaLIB は Vault の内容、メディアソースのパス、スキャン中のプライベートなファイル名を表示しません。また、ホーム、続きを見る、視聴済み、お気に入り、見たい項目にも Vault の作品は出ません。ロック解除後は、通常の項目と同じように閲覧、再生、お気に入り、マーク、再生履歴の削除ができます。

### ライブラリヘルスとタスクセンター

ライブラリヘルスでは、メディアソースのオフライン、無効なローカルパス、リモート再生パスの欠落、重複の可能性、カバー画像や年の不足などを確認できます。すべての整理操作は確認を求め、削除するのは MediaLIB の内部索引だけです。元のメディアファイルは削除されません。

<div align="center">
  <img src="./assets/task-center.png" alt="タスクセンター" width="86%" />
</div>

タスクセンターには、スキャン、差分スキャン、リモート同期、カバー画像の事前読み込み、メタデータ補完、動画キャッシュ、チャプター画像、一括整理などのバックグラウンドタスクが記録されます。失敗したタスクでも、対象を再構築できる場合は再試行ボタンが表示されます。

## クイックスタート

1. `MediaLIB.app` を開きます。
2. 「メディアソース」に入り、「メディアソースを追加…」をクリックします。
3. ローカルフォルダ、ネットワークデバイス、Emby、Jellyfin、Plex のいずれかを選びます。
4. ローカルまたはネットワークフォルダの場合は分類を選びます。迷ったら「自動識別」を使います。
5. 「すべてスキャン」をクリックするか、個別のメディアソースをスキャンします。
6. スキャンが終わったら、サイドバーから「ホーム」「ビデオ」「音楽」に入って閲覧します。
7. オンライン情報を補完したい場合は、「設定 > メタデータとマッチング」で対応するデータソースを設定します。

初めて使うときは、まず小さめのフォルダで試すのがおすすめです。分類やカバー画像の結果を確認してから、完整なメディアライブラリを追加すると安心です。

## インストール

`MediaLib.dmg` を受け取った場合：

1. DMG を開きます。
2. `MediaLIB.app` を「アプリケーション」にドラッグします。
3. 未署名ビルドを初めて開くとき、macOS にブロックされる場合があります。`MediaLIB.app` を右クリックし、「開く」を選び、もう一度確認してください。

動作要件：

- macOS 13 Ventura 以降。
- ローカルスキャンと再生にはアカウントは不要です。
- TMDB、音楽メタデータ、リモートサーバー、字幕ダウンロード、Trakt、Last.fm などの機能には、ネットワーク接続と対応するサービスのアカウントまたは API Key が必要です。

## データの安全性

MediaLIB の基本方針は、メディアファイルを勝手に変更しないことです。

- スキャンは索引を作るだけです。
- 再分類は MediaLIB 内部の分類だけを変更します。
- お気に入り、見たい、評価、再生履歴、プレイリストはローカル索引に保存されます。
- 無効な索引を整理しても、元のファイルは削除されません。
- オフラインキャッシュの削除は、MediaLIB が作成したキャッシュコピーだけを削除します。
- Vault がロックされている間は、パス、ファイル名、内容が隠されます。
- データベースのバックアップと復元は、MediaLIB の内部データだけを扱います。メディアソース内のファイルをコピーしたり置き換えたりすることはありません。

## プロジェクトの状態

MediaLIB は現在も継続して磨き込み中です。今は、メディアライブラリ、プレイヤー、リモート同期、音楽体験、オフラインキャッシュ、画面のパフォーマンスをより安定して使いやすくすることに重点を置いています。

使っていて問題があれば、Issue で知らせてください。ページ、操作手順、スクリーンショット、ログを添えてもらえると、原因を見つけやすくなります。

## ライセンス

このリポジトリは現在、個人の学習と利用を目的としています。App に同梱、または利用される libmpv、ffmpeg などのサードパーティコンポーネントは、それぞれのオープンソースライセンスに従います。再配布や商用利用を予定している場合は、先に関連するライセンス要件を確認してください。
