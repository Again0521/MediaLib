# MediaLIB 用户使用说明

文档状态：2026-06-10 更新。

1. 打开 `MediaLIB.app`。
2. 进入“媒体源”，点击“添加媒体源…”。在向导第一步选择“本地目录”“网络设备”“Emby”“Jellyfin”或“Plex”；本地目录可选择本机视频目录、移动硬盘目录，网络设备会在 macOS 挂载后选择 `/Volumes` 下的真实目录。
3. 本地或网络目录会在向导第二步选择目录类型。例如电影目录选“电影”，动漫目录选“动漫”，电视剧目录选“电视剧”，音乐目录选“音乐”；私密目录选“保险库”，无法确定时可保留“自动识别”。第三步会确认这个来源是否参与元数据拉取、是否参与健康检查；本地/网络来源还可选择是否把可写元数据优先写到源目录，远程来源可选择痕迹同步策略。重复添加同一个本机或网络挂载目录时，MediaLIB 会提示该媒体源已存在；保险库来源的重复提示不会显示目录名。
4. 点击“扫描全部”或单个媒体源旁的扫描按钮。
5. 扫描完成后，视频内容会在左侧“视频”分组下按电影、电视剧、动漫等分类显示；视频“喜欢”表达偏好，“想看”表达未来观看计划，两者互不影响。本地、Emby / Jellyfin / Plex 和保险库视频都可标记想看或喜欢，但入口彼此隔离：本地视频只出现在“视频”目录，远程服务器想看出现在对应 EMBY、Jellyfin 或 Plex 目录，保险库内容只在解锁后的保险库页面显示。视频侧栏底部可新建智能集合和手动集合：智能集合可按媒体类型、状态、最近加入时间、年份、资料源评分、用户评级、题材关键词和来源保存动态筛选规则，并支持“满足全部 / 满足任一”；可在集合页面点击“编辑规则”修改。手动集合由你在海报、详情封面、剧集行或批量选择栏中加入/移出条目，适合整理专题片单。本地视频集合只接收本地、挂载来源和解锁后的保险库视频，不会把 Emby / Jellyfin / Plex 条目混入本地“视频”目录；远程内容保持在各自远程目录中，只在首页聚合展示。选择“新建集合并加入”时会先让你命名集合，再把当前条目加入；进入手动集合页后默认按加入顺序展示，也可以从排序菜单临时切换为标题、年份、评分等查看方式；在集合页右键单个条目或批量选择后，可通过“调整集合顺序”置顶、上移、下移或移到底部。手动集合和智能集合都可以在编辑面板打开“首页展示”，也可以在集合页页头或侧栏右键选择“发布到首页”；发布后会作为首页总览横向看板显示，默认关闭，且首页只展示公开视频与远程服务器顶层内容，不会把保险库内容带入总览。侧栏和集合页标题旁的集合封面会自动使用集合内前几项海报拼贴生成，空集合会显示统一集合图标。集合只保存 MediaLIB 内部规则或条目 ID，不复制或移动媒体文件。音乐内容会在“音乐”分组下按歌曲、专辑、艺术家、歌单、最近播放等方式浏览，“未匹配歌曲”不再作为左侧栏栏目。收藏的歌曲会作为“歌单”页最上方的固定歌单显示，不会被删除；进入后可播放、查看并通过移出歌曲来取消收藏。点击专辑卡片、艺术家行或歌单卡片可以进入对应歌曲列表；右键或点击播放某个专辑、艺术家或歌单时，会把当前音乐队列替换为这个分组里的歌曲。自动识别媒体源会穿透文件夹识别音频和视频，普通歌曲不会被视频文件大小阈值跳过；分类页扫描也会包含自动识别来源，由文件名和剧集文件夹结构继续归类。
- **片库健康中心**：点击首页健康提示或左侧“管理 > 片库健康”进入，可集中查看离线媒体源、失效文件/播放路径、疑似重复条目和缺少封面/年份/简介/音乐标签的内容，并重新挂载、扫描、打开详情核对或补充元数据。本地视频文件不存在会进入失效文件路径；参与健康检查的 Emby/Jellyfin/Plex 视频如果没有可播放路径，也会作为失效播放路径显示为“远程播放路径未记录”。点击“一键补充”后会把视频和音乐元数据补充加入任务中心，只补空缺、不覆盖已有信息；在详情页搜索并应用元数据结果时，弹窗会立即关闭，封面下载和索引写入完成或失败后会在前台用排队的软件内通知反馈，MediaLIB 不在前台时优先使用系统通知，未授权时回到应用后补发。“移出索引”和“清理失效索引”都必须确认，操作只删除 MediaLIB 内部索引，不会删除、移动或修改媒体文件；暂时离线来源中的条目不会被清理，保险库锁定时不会显示其路径、内容或来源名称。
- **来源设置**：每个媒体源行都有设置按钮。点击后可在弹窗中调整本地/网络来源分类、元数据拉取、健康检查、写入源目录偏好，或调整 Emby / Jellyfin / Plex 的服务器库选择和痕迹同步策略。点击保存后弹窗会立即关闭，保存结果或远程库刷新结果会通过任务通知反馈；MediaLIB 不在前台时优先使用系统通知，未授权时回到应用后补发软件内通知。关闭健康检查或删除来源后，该来源已有的离线、失效路径、重复和元数据缺口结果会从健康中心移除。设置只修改 MediaLIB 内部索引和来源偏好，不移动、删除或重命名媒体文件。
- **歌单与保险库播放记录**：“新建智能歌单”位于“音乐 > 歌单”页面。保险库解锁后，其中有播放痕迹的视频会出现在“正在观看”或“已观看”；可使用页面右上角“清除记录”，也可在保险库海报上右键清除单条记录。锁定保险库后这些内容会再次隐藏。
- **控件反馈与下拉菜单**：页头扫描、清除记录、设置页操作、媒体源行按钮、批量操作栏和返回顶部按钮在鼠标经过和点按时会显示边缘光、描边、亮度或透明度反馈；筛选胶囊和首页标签也会在鼠标经过时变亮。选择菜单会根据当前选项自动调整宽度，完整选项仍在弹出菜单中显示。
- **系统文件选择框**：添加媒体源、选择挂载目录、导入/导出歌单、选择外部播放器、恢复备份和选择自定义封面时使用 macOS 原生不透明文件面板，不会再透出后方页面内容。
- **按钮与弹窗**：打开后续流程的按钮会带省略号，例如“添加媒体源…”和“选择…”。立即执行的动作不会带省略号，例如“扫描全部”和“保存”。普通弹窗使用一致的标题、说明和底部按钮布局。
- **浮窗通知与页面提示**：任务加入、完成、失败、取消，以及加入喜欢、加入想看、更新评级、标记已观看、删除播放记录等轻量状态，会以顶部中央胶囊浮窗显示，几秒后自动收起，也可以手动关闭。软件内通知会排队逐条显示，前一条消失后才显示下一条，不会多条叠在一起。短提示会按文字长度收窄，较长的片名、路径或任务说明会在浮窗内居中换行，不会把整条通知固定撑宽；通知、页面提示、弹窗提示卡、行内说明和播放器提示的图标统一使用 MediaLIB 主配色。后台任务完成时，如果 MediaLIB 不在前台，会优先使用 macOS 系统通知；如果系统通知未授权或不可用，回到 MediaLIB 后会补发软件内通知。进入首页、任务中心、媒体源、片库健康或音乐歌曲列表时，MediaLIB 可能用同样的浮窗给出一条当前页面提示；同一条提示只会在首次进入时显示。
- **自动增量扫描与任务中心**：在“设置 > 媒体库更新”开启自动扫描后，本机来源中的文件新增、修改和删除会先短暂合并，再只更新受影响条目；目录删除、目录重命名或系统文件事件不完整时会自动执行安全完整扫描。移动硬盘、SMB/FTP/NAS 等来源继续按设置间隔扫描，离线时不会被当作文件删除。左侧“管理 > 任务中心”可查看完整扫描、增量扫描、Emby/Jellyfin/Plex 同步、封面预热、元数据补充、视频缓存、章节图和一键清理状态，取消本地扫描，或清理已完成记录；任务记录会保留到下次启动，重启时未完成任务会标记为中断。失败任务如果仍能从当前媒体源或媒体条目重建目标，会显示重试按钮：完整扫描、增量扫描、远程同步、封面预热、一键清理和元数据补充可直接重试；增量扫描重试会改为该来源的安全完整扫描；新产生的视频缓存、章节图和片头片尾检测任务也会记录媒体目标，失败后可重试。旧历史任务缺少目标上下文时不会显示重试按钮。视频缓存任务会显示进度和已下载大小，并可暂停、继续或取消；章节图任务会显示预览图生成进度，并可取消。视频缓存位置和容量上限可在“设置 > 视频播放与缓存”中调整，MediaLIB 会在所选目录下建立 `VideoCache` 子目录；一键清理会按容量策略回收已看或较久未播放的超限缓存。保险库来源添加后只显示正常扫描任务反馈；保险库任务、离线来源、删除确认和失败提示只显示通用说明，不展示条目名、来源名、路径和文件名。
- **章节与跳过片头/片尾**：使用内置视频播放器时，点击控制条左侧的书签按钮可查看视频内嵌章节，也可把当前播放位置设为片头开始/结束、片尾开始/结束，或添加手动章节。时间轴上的刻度可帮助定位；进入已完整标记的片头或片尾范围后，播放器会显示“跳过片头/片尾”按钮。视频或系列右键可选择“检测片头片尾”，MediaLIB 会在任务中心分析内嵌章节关键词并生成待审核自动标记；待审核标记会出现在播放器章节弹层中，只有点击确认后才参与跳过，忽略后不会再作为普通标记显示。视频或系列右键也可选择“预生成章节图”，MediaLIB 会按有限时间段把进度条悬浮预览帧写入内部缓存；之后拖动或悬停进度条时会优先显示这些预览图。手动标记可在同一弹层删除，所有标记和章节图缓存只保存在 MediaLIB 索引/缓存中，不修改视频文件。
6. 点击海报进入详情页，播放单文件、剧集或 Emby / Jellyfin / Plex 流媒体后会保留当前详情页与选中位置；点击左上角“返回”或按 Esc 返回列表。详情页应用 TMDB/音乐信息、音乐标签工作台或一键补充后，如果最近一次覆盖已有历史，会出现“撤销元数据”按钮，可恢复最近一次覆盖前的内部索引字段；设置“连接器与同步”里的元数据历史也可按批次撤销。撤销不会移动媒体文件，也不会撤销用户显式写入原音乐文件标签的文件级变化。
7. 电影可直接播放；电视剧会显示剧集列表，单击某一集会选中，双击会直接播放，也可右键播放、快速预览或用外部播放器打开。
8. 视频会使用随 App 分发的 libmpv 内核在 MediaLIB 自己的透明标题栏窗口内播放，窗口会按设置中的屏幕宽度百分比、视频比例和控制条安全空间自动调整大小，并限制最小宽高，避免清晰度、音轨、字幕、倍速、全屏和锁定按钮在缩小时错位；保留 macOS 原生红黄绿按钮和系统圆角，红黄绿按钮会随控制层自动淡入淡出。支持 MKV 等常见格式，播放器窗口可拖动调整大小，也可点击全屏按钮切换全屏。播放器会自动尝试加载视频同目录同名字幕，字幕和音轨可从播放器内图标菜单切换，不会再额外打开独立 mpv 视频窗口或 Dock 图标；关闭或切换上下集时会先停止当前播放核心。
9. 视频播放窗口使用底部紧凑液态玻璃悬浮控制条，鼠标移入时显示，播放时会自动收起；控制条底色更均匀，右键菜单、齿轮设置、音轨/字幕/画质/章节弹层、预览气泡、音量提示、加载/错误提示和截图反馈会随当前播放器明暗策略统一调整文字和图标颜色。播放画面右键只保留更适合作为上下文动作的“从头播放”“保存当前画面”和“用系统播放器打开”，字幕、音轨、清晰度、倍速、章节、全屏、上一集/下一集等已在控制条有入口的功能不再重复出现在右键菜单。截图会保存到用户图片目录下的 `MediaLIB 截图` 文件夹，不会修改视频文件。macOS 顶部“播放”菜单和系统媒体键也支持基础播放控制。弱网播放时播放器会使用内存缓存和预读来降低卡顿；“画质”按钮本身只显示文字，具体 1080P/2K/4K 档位在弹出列表中选择。控制条齿轮弹层分为“当前视频”“窗口与时间轴”和“完整设置”：当前视频可开启单片循环、设置或清除 A-B 循环并打开播放信息；窗口与时间轴可调整窗口宽度、窗口置顶、剩余时间显示和预览图模式；点击“打开内置播放器设置”会在播放器中央打开完整设置窗口，可调整默认倍速、快进/快退、恢复播放自动回退、片头片尾跳过策略、记忆进度、自动下一集、自动标记已看、音频延迟、字幕延迟、字幕大小、字幕位置、画面比例、黑边裁切、去隔行、画面旋转、播放器快捷键、双击全屏、鼠标滚轮音量和触控板手势。视频默认使用内置还是系统播放器只在“设置 > 视频播放与缓存”的启动方式中调整，不出现在播放器窗口设置里。播放信息只展示标题、时长、分辨率、编码、当前轨道、清晰度和调校状态，不显示本地路径或远程 token URL。触控板横扫可按设置跳转，竖扫可调音量，捏合全屏默认关闭并可手动开启；普通鼠标滚轮可在鼠标操作中开启为音量控制，不会改变播放进度。切换远程服务器清晰度时会尽量保留当前画面和播放状态，按当前时间续播；进度条拖动时先预览位置、松手后再提交跳转，减少清晰度切换后拖动进度条被拉回 00:00 的情况。默认支持 Space/K 播放暂停、方向键跳转和调音量、M 静音、F/Return/Cmd-F 全屏、数字键比例跳转、`[`/`]` 调整倍速、`,`/`.` 逐帧、Z/Shift-Z 调整字幕延迟、Control--/Control-= 调整音频延迟、G/Shift-G 调整字幕大小、R/Shift-R 调整字幕位置、D 切换去隔行、C/V/A 切换字幕或音轨、B 设置 A-B 循环点、Shift-B 清除 A-B 循环、I 打开播放信息；这些视频窗口内快捷键可在完整设置窗口中逐项记录、清除或恢复默认。可通过控制条左侧关闭按钮、Esc、Cmd-W 或 Q 退出，关闭播放器不会退出主应用。
10. 音乐条目双击或点击播放后会优先进入底部迷你播放器，而不是直接展开整页播放界面；点击底栏曲目信息后才会展开铺满 MediaLIB 主界面的音乐播放页。展开页保留 macOS 系统红黄绿按钮，点击系统关闭按钮会退出应用，同时隐藏主窗口工具栏、标题和侧栏展开按钮；左上角固定悬浮的收起按钮只负责回到底部迷你播放器。展开过程会先挂载整窗播放器覆盖层，再延后切入沉浸窗口 chrome，收起时先回到底栏再恢复 chrome，外层只做透明度转场，不再缩放整窗覆盖层，因此顶部不应再露出系统白底；展开时下层媒体库会停止响应鼠标 hover 和点击，避免被遮住的列表继续触发光效。展开页左侧封面、长标题和控制栏会按窗口高度自适应；右侧歌词卡片会弹性伸缩并保留合理最小宽度，窗口较小时优先保证左侧控制栏宽度和左边距，再让歌词卡片从右边界向左收窄。暂停/播放时，专辑封面会用连续动画同步调整后退距离、缩放、阴影和光晕，不会出现先关光再缩小的两段感。展开页控制栏第一行依次为喜欢、弹性进度条和队列，第二行依次为隔空播放、音量、上一首、播放/暂停、下一首、随机播放和循环模式；第一行首尾按钮与第二行首尾按钮对齐，进度条按剩余空间自动拉伸；播放/暂停使用和底部条一致的蓝色胶囊主按钮；循环模式是一个按钮，会在顺序播放、队列循环和单曲循环之间切换，不再显示快退/快进 15 秒按钮。底部迷你播放器会贴在主界面底部并跟随侧栏弹性变化，循环模式按钮位于随机播放左侧，队列按钮位于隔空播放左侧，并已移除快退/快进 15 秒按钮；当底栏和列表/设置页同时存在时，任意方向滚动都会把底栏平滑收起到右侧，只保留变暗的专辑封面、低开销真实频谱和位于封面外侧的播放进度环，浅色表示未播放、深色表示已播放，点击封面可展开完整底栏。隔空播放按钮使用固定深蓝色，不跟随专辑封面取色；展开页和底部条由可见按钮主动触发系统原生隔空播放选择器，因此展开/收起切换后仍可点击。隔空播放状态会在展开页和底部条之间保持，音乐隔空播放遵循系统外部路线逻辑，不再提供本机同播开关。收藏会即时刷新界面并在后台保存，收藏集合可在“音乐 > 歌单”的置顶歌单中管理。歌曲右键可加入队列、下一首播放或添加到歌单；“添加到歌单”弹出菜单第一项是新建歌单，后面是已有用户歌单。队列弹层可把当前队列一键存入新建或已有歌单，也可右键单首歌曲加入歌单；队列顺序、随机开关和循环模式会在退出后保存，并在下次启动恢复，但软件不会因此自动播放或主动显示底栏。底部条和展开页打开队列时会在本次启动内共用上次滚动位置；队列列表使用固定行高和小尺寸封面缓存，长队列滑动时不会让每一行持续回报几何位置。歌单只保存 MediaLIB 内部歌曲索引和顺序，不移动、不改名、不复制音乐文件。底部迷你播放器里的关闭按钮会结束当前播放。上一曲/下一曲会同步切换封面、实际声音和界面取色，且音乐每次播放和切歌都会从头开始，不再记住上次播放到的位置；切歌时会保留旧取色和纹理直到新封面取色与颜料场就绪，避免播放器先闪成灰底。底部条本体是透明白色厚玻璃，专辑封面会在左侧完整显示，内部裁剪从封面向右衰减的低成本专辑色柔光，并给右侧按钮形成由近到远递减的轮廓光；底栏常驻进度条改为轻量绘制控件，不再热更新原生 slider，因此设置页和长列表叠着底栏时更稳。视频和音乐会分别记住上次调节的音量。优先使用音乐文件内嵌封面和标签，扫描会读取 common metadata 以及 iTunes/ID3/QuickTime 等格式级标签，没有内嵌封面时等待联网补全或使用蓝青系默认音乐封面；联网封面不会覆盖内嵌封面，内嵌歌词也可在展开播放页显示。扫描会忽略歌词、字幕、本地图片、`.cue` 和 `.nfo` 等旁路文件，只把真实音频/视频文件入库，并按规范化路径去重。
- **响度均衡与跨曲过渡**：进入“设置 > 音乐播放”，可选择关闭、按歌曲均衡或按专辑均衡。MediaLIB 只读取歌曲已有的 ReplayGain / R128 标签并在播放时使用峰值保护调整音量，不会重新编码、写回或修改音乐文件；没有响度标签的歌曲保持原始音量。跨曲过渡默认保持即时衔接，本地顺序播放和队列循环会提前准备确定的下一首并由同一播放器连续切换；也可选择柔和淡入并调整时长。
11. 如歌曲旁边存在同名 `.lrc` 或 `.txt` 歌词文件，或音频文件内嵌歌词，会在全屏播放页右侧歌词区显示；目录级 `lyrics.lrc` / `lyrics.txt` 不会被当作整个文件夹的共享歌词。音乐页底层是专辑封面多色取色和低频封面颜料场合成的不透明 Metal 色板背景，不会透出下层媒体库；颜色会随切歌平滑更新，右侧歌词卡片、控制栏和收起按钮使用同一套更透亮的中性圆角液态玻璃透镜，像无色玻璃浮在底层专辑色板之上；玻璃底材质不改成专辑色实底，但边缘、上沿和中心舞台光会接受方向化专辑光照，白色只保留很薄的镜面高光。专辑封面会以封面中心为锚点向外形成克制范围的多色径向柔光和斜向光束，封面下沿还会投射一层更宽的预烘焙封面复制光；这些专辑色主要存在于底层背景、封面周围和前景玻璃受光层，不直接污染玻璃底材质。底部条和队列弹层不再跟随鼠标产生光效，只保留静态玻璃、专辑柔光和低成本轮廓。有歌词时不会显示获取歌词按钮；歌词面板右下角会用低存在感标记区分“原词逐字”“音频对齐”“估算同步”。带时间戳的 LRC 会跟随播放器回读的真实歌词时钟弹性滚动，进度条拖动时控制条可以立即显示目标位置，但歌词当前行和逐字高亮会等待播放器真实时钟稳定后再校正；如果真实落点比目标靠前或靠后几行，歌词卡片会平滑滚回实际当前行。恢复期间当前行会临时整行高亮，避免旧时间戳驱动逐字高亮时出现字号或换行错乱，行尾后再恢复逐字特效。第一行歌词从卡片中心出现，当前歌词行保持在中心附近并柔和变色、加粗、轻微放大，未播放的字会略微靠下，播放到对应字或增强 LRC 片段时会缓慢上升并稍微突出，不再使用呼吸式柔光；如果歌词自带 TTML、YRC、QRC、KRC 或增强 LRC 的逐字/片段时间戳，会优先使用真实时间；如果只有普通逐句 LRC，会先按当前行到下一行的时长估算当前字，中文按字符、英文按单词，空格和标点不参与计时，最后一句会用前几句平均或保守时长兜底；本地音频文件可读时，MediaLIB 会在后台分析当前歌词行对应音频片段的人声能量、静音区间和峰值变化，生成并缓存更精细的逐字对齐结果，完成后无缝切换。拖动进度条或手动拖拽歌词后，歌词会用目标时间落入的歌词时间区间定位当前句，也就是目标时间位于哪一句的开始时间和下一句开始时间之间，就居中高亮哪一句；下一条时间戳只作为当前句结束边界。远离当前行的歌词会逐步变淡并被玻璃模糊遮挡；当一句歌词在卡片内自动换成多行时，高亮会按实际换行后的字符顺序推进，不会多行一起横向渐变，换行和当前行切换也会更缓和。手动拖拽歌词区可临时查看其他行歌词，此时远离当前行的歌词不会再被模糊，当前行轻动效也会暂停；停止浏览后会保留约 4 秒清晰阅读窗口，再恢复自动居中与景深；点击带时间戳的歌词行可跳转到对应位置。歌词滚动时不会显示滚动条。无时间戳歌词会显示普通可滚动文本，同样不显示滚动条。
- 2026-06-09 更新：音乐展开页的封面光现在除封面近场光外，还会在页面底层叠加一层预烘焙封面复制光；歌词卡片、控制栏和收起按钮保持中性液态玻璃底材质，但会被专辑色外部光照亮边缘、上沿和中心舞台区域，白色只作为薄镜面高光，不再把专辑色浸染成前景玻璃实底。第二轮继续降低黑色和近黑色在发光中的权重，黑色区域只提供深度和阴影，不会向外发光；底板会把封面低频颜色柔化、清洁并轻微转向，减少过艳、刺眼、灰脏或看起来不像来自封面的情况。控制栏高度固定，不会因不同封面颜色被拉高。
- 2026-06-09 晚间更新：如果 LRC 中同一个时间戳同时包含日文原文和中文翻译，MediaLIB 会把它们保留为相邻的独立歌词行，不再合并成一行显示；显示顺序沿用歌词文件本身，但播放和自动居中会优先锚定含假名的日文原文，翻译行保持低模糊伴随显示。歌词卡片上下边缘的磨砂和景深略增强，中心歌词行下方增加更柔和的专辑色舞台光，中间区域更通透；控制栏和收起按钮也降低灰雾感并提亮边缘。底板颜色在保持克制和干净的基础上微微提高饱和度。
12. 在媒体源页点击“添加媒体源…”，第一步选择 Emby、Jellyfin 或 Plex。Emby/Jellyfin 第二步输入服务器地址、用户名和密码；Plex 第二步输入服务器地址和 Plex Token。第三步选择这个来源的元数据拉取、健康检查和痕迹同步方式，然后点击“登录并同步”或“连接并同步”。向导会立即关闭，认证与同步在后台继续，可在任务中心查看进度；完成或失败后会通过对话框反馈。同步成功后，左侧栏会显示独立 EMBY、Jellyfin 或 Plex 目录，不会混入本地“视频”或“音乐”目录。远程 token、自动恢复登录所需凭据和网络设备凭据保存在 MediaLIB 的本机受限配置目录中，不再读写或删除系统钥匙串，因此更新软件后首次打开不应再弹系统级钥匙串密码框。Emby/Jellyfin 新连接在 token 失效时会自动重新认证；Plex Token 失效时会提示重新连接。媒体源行的设置按钮可选择要同步的服务器媒体库，默认同步全部；改为指定库并保存后会立即刷新远程目录，取消勾选只会从 MediaLIB 内部索引移除对应内容，不会改动服务器。展开 EMBY、Jellyfin 或 Plex 后，MediaLIB 会按有内容的视频、音乐、最近播放、想看、收藏，以及服务端媒体库分类自动显示入口，无条目的分类不会显示。设置弹窗也可选择“数据双向同步”“仅从服务器同步”或“数据不同步”：双向同步会互相更新播放记录和已观看状态，Emby/Jellyfin 也会同步收藏；Plex 喜欢状态第一阶段仅保存在 MediaLIB 本机。单向同步只接收远端状态，不把本机痕迹写回服务器；不同步会尽量保留本机播放记录、收藏、已观看和评级，适合多人共用同一账号的场景。本机“想看”不会写回服务器，但会在对应远程目录下单独显示。重新同步时会导入服务端分类/类型标签，并在完成后后台预热远程海报。远程剧集会在详情页正常显示剧集列表，简介下方也会显示分类标签；远程视频和音乐可像本地媒体一样使用内置播放器或系统播放器播放，不需要本地文件路径。详情页会显示“Emby 流媒体”“Jellyfin 流媒体”或“Plex 流媒体”，不会直接展示带 token 的播放链接；远程海报会缓存到 MediaLIB 内部，切换标签、滚动或鼠标经过海报时不应再闪回默认封面。
13. 在统一向导第一步选择“网络设备”时，第二步输入 `smb://`、`ftp://` 或 `ftps://` 地址，选择匿名或账号密码登录，并在同一步选择分类；第三步确认元数据拉取、健康检查和写入源目录偏好，macOS 挂载后选择真实目录并开始扫描。NAS 或远程设备重启导致 macOS 断开挂载后，MediaLIB 在手动/自动扫描前会尝试用保存的网络地址重新触发系统挂载；离线网络来源行也会显示“重新挂载”按钮，可手动触发同一套系统挂载流程。如果网络、共享名或账号已经变化，仍需要重新登录或手动挂载。
13. 剧集列表中选中剧集后按空格可快速预览，Esc 或再次关闭窗口可退出。
14. 设置页按用途分成“首页入口”“外观与布局”“视频播放与缓存”“音乐播放”“媒体库更新”“元数据与匹配”“字幕下载”“封面与截图”“账号与同步状态”“Trakt 同步”“保险库”“数据、存储与诊断”和“关于软件”。其中可分别调整视频默认启动方式、视频系统播放器路径、已看判定、视频缓存位置和容量上限、音乐播放器、歌词同步、响度均衡、跨曲过渡、首页选项卡、自动扫描、缺失封面处理、截图封面位置、海报尺寸、保险库密码、系统播放器路径等；视频窗口宽度、窗口置顶、剩余时间、预览图模式、默认倍速、跳转间隔、恢复播放回退、片头片尾跳过策略和鼠标/触控板输入习惯在内置视频播放器齿轮菜单和完整设置中调整；当前视频的 A-B 循环和单片循环只在播放器齿轮弹层中操作。Trakt 分组可连接设备码授权账号，之后本机标记已看/想看会自动推送到 Trakt；手动标记已观看时，如果条目仍在想看中，也会同步移出本机想看和可表达的 Trakt watchlist；也可点击“导入状态”从 Trakt 拉取 watched/watchlist 差异并生成同步冲突，导入完成/失败会按任务通知反馈。“账号与同步状态”会显示远程账号、待处理同步冲突和可撤销元数据历史数量，可打开冲突队列记录“保留本地 / 采用远端 / 合并 / 都保留 / 忽略”的处理选择；其中“采用远端”会把已看、想看、喜欢和用户评级等可安全表达的状态写入 MediaLIB 内部索引，采用远端已看时也会从本机想看移出，Trakt watched/watchlist 冲突选择“保留本地”会把本机状态写回 Trakt，写回结果同样按任务通知反馈。上述操作不会修改媒体文件；Plex/Jellyfin/Emby 的更多远端写回、评分冲突来源接入和失败恢复仍在后续连接器队列中，也可打开元数据历史按批次撤销。当前不提供本地多档案，播放进度、播放次数、已看、喜欢、想看和评分都使用全局本机索引。Jellyfin 和 Plex 已可通过媒体源页统一添加向导第一阶段登录/连接同步；iCloud 的真实同步仍需后续接入协议，当前不会伪装成可用连接。“数据、存储与诊断”中可查看数据库版本、立即创建一致性备份、打开备份位置、从备份恢复，或执行“一键清理”。一键清理只整理 MediaLIB 内部缓存和历史：失效媒体源缓存、失效剧集缓存、孤儿缓存文件、超出容量上限的离线视频和过长任务记录，不会删除、移动或修改媒体源里的文件。恢复会先自动保存当前数据库并停止正在播放的媒体和扫描任务，只替换 MediaLIB 内部索引、播放记录、喜欢、想看、视频集合、歌单、队列和同步状态，不会复制、移动或修改媒体文件。保险库密码框、统一媒体源添加向导和元数据搜索框也使用同一套暖白玻璃输入样式。
15. 缺少本地海报时，可选择“截取视频帧”保留视频真实比例，也可选择“绘制默认封面”生成统一风格封面；单视频文件在抽帧或默认封面路径下会优先按视频分辨率显示为横版比例，避免初次显示成竖版海报。MKV 等 AVFoundation 无法截图的格式会尝试使用 App 内打包的 `ffmpeg` 兜底截帧。音乐缺省封面会使用蓝青系玻璃唱片/音符图形。
16. 首页选项卡可在设置中开启或关闭，包括总览、下一集、继续观看、离线、分类、喜欢、未观看和保险库入口；首页视频看板会同时包含本地公开视频和远程服务器顶层视频，没有内容的分类不会出现在首页或首页设置中，离线标签只在存在可播放缓存时显示，总览统计也会隐藏 0 项卡片。总览统计卡会按窗口宽度弹性占满所在列并保持对齐。总览页会展示继续观看、下一集、为你精选、同题材高分、高分剧集、想看清单、近期高分、离线可看和已发布集合看板，有内容的看板会优先靠上显示；看板会缓慢自动横向滚动，鼠标悬停、触控板横向滑动或按住鼠标拖动时以你的操作为准。总览看板里的视频、系列和剧集卡片也支持右键菜单，菜单动作与海报墙一致；本地条目可重新分类，Emby/Jellyfin/Plex 条目不会出现本地重分类入口。“继续观看”和“下一集”看板可右键删除播放记录，下一集会清除所属系列播放痕迹以移除后续提示。拉伸主窗口宽度或用系统按钮收起/展开侧边栏时，总览看板会暂时停下自动滚动和 hover 光效，避免横向看板参与高频布局，窗口稳定后会自动恢复。推荐会参考你看过、继续观看、喜欢、想看和标星的题材，也会结合资料源评分，优先把评分较高且还没看完的剧集或系列排到前面；调整已看阈值后，总览推荐和未看/已看判断会立即按新阈值刷新；内容不足时会回落到库内其他高分视频。首页标签栏会先保持单行，只有当前窗口宽度放不下时才自动切成可拖动的双行标签栏，右侧入口不会被窗口挡住；“最近添加”只是浏览入口，不会出现清除记录按钮。首页、视频分类、EMBY/Jellyfin/Plex 页面和音乐页面里的“扫描”会按当前分类扫描对应媒体源；在某个远程来源或远程服务器媒体库中扫描时，只会刷新当前 Emby / Jellyfin / Plex 来源，媒体源页的“扫描全部”才会全量扫描。开启“设置 > 媒体库更新 > 自动扫描”后，MediaLIB 会定时扫描已添加且可访问的路径，发现新媒体；网络设备断开后重连，可等待自动扫描或手动点当前分类“扫描”恢复索引和本地封面。首页上次选择的标签、侧边栏位置、媒体库筛选、排序项和排序方向会自动记住；再次选择当前排序项会在正序/倒序之间切换，并在需要重算快照时显示和初次进入一致的加载占位。
17. 保险库分类第一次进入会引导创建 4 到 8 位数字密码；PIN 哈希保存在本机 MediaLIB 配置目录中，不再触碰系统钥匙串。之后可使用 Touch ID 或手动输入密码解锁，锁定状态会隐藏保险库路径、扫描文件名和改密入口。保险库名称可在设置中重命名。
18. 右键视频海报或首页总览卡片可播放、删除播放记录、加入/移出想看、喜欢/取消喜欢、手动标记已观看/未观看（标记已观看会自动移出想看）、重新分类、加入集合、从当前集合移除、调整当前集合顺序、预生成章节图、缓存远程视频到本地、删除缓存文件或设置 1 到 5 星评级；Emby/Jellyfin/Plex 条目不会显示本地“重新分类”。剧集列表也可右键加入集合、预生成章节图、缓存或删除单集缓存。远程服务器视频存在清晰度选项时，缓存前可以选择原画、1080P、2K 或 4K 等可用档位；重新缓存同一视频前会先删除旧缓存副本。系列海报右键还可使用“自动缓存系列”，选择自动缓存下一集、未看 3/5/10 集、自定义未看集数、当前整季或全系列；有多档清晰度时，可在“指定缓存清晰度”中为自动缓存选择固定档位。已开启的规则可暂停 7 天、继续，设置不自动到期或 7/30/90 天后到期，切换“允许远程网络 / 仅本机局域网 / 仅 Wi-Fi”策略，或用“停止自动缓存”关闭；到期只关闭 MediaLIB 内部自动缓存规则，不删除已缓存视频；仅本机局域网会只排入 localhost、`.local` 和私有网段地址的远程剧集，仅 Wi-Fi 会在当前系统网络路径使用 Wi-Fi 时才排入新的自动缓存任务。自动缓存会把需要补齐的剧集加入普通视频缓存任务，仍可在任务中心查看进度、暂停、继续或取消。缓存完成后，剧集封面会标注“已缓存”，系列海报会按情况标注“部分已缓存”或“已全部缓存”，首页“离线”标签也会显示这些可播放缓存；在系列海报上删除缓存会删除该系列已缓存剧集，在单集上删除缓存只会删除该集。设置页会显示当前离线视频缓存占用，设置容量上限后，一键清理会优先保留最近播放的缓存，并回收已看或较久未播放的超限缓存。原画缓存会保留视频容器里的音轨，远程服务器文本字幕会尽量同步为同名旁路字幕；非原画转码档位的音轨保留情况以服务器转码输出为准。海报墙、详情页主封面、剧集行封面和音乐专辑封面在鼠标经过时会随鼠标位置轻微倾斜，方便检视封面，视频和剧集海报卡片还会轻微放大。视频左侧栏不再提供独立“最近播放”页，播放过或手动标记过的视频统一进入“正在观看”，在该页可一键清除当前播放记录；首页“下一集”卡片右键删除播放记录时会清除所属系列的播放痕迹。音乐歌曲列表和剧集列表中也可右键删除播放记录。删除播放记录只清除播放位置、进度、已观看和最近播放时间，不会删除媒体条目、清空音乐列表或要求重新扫描。重分类、集合整理、自动缓存订阅和章节图生成只修改 MediaLIB 内部索引/缓存，不会移动物理文件。
19. 在设置的“元数据”区填写 TMDB API，详情页可搜索并应用电影/剧集信息；当设置语言和已有 TMDB 元数据语言不一致时，一键拉取会使用当前设置语言重新拉取并覆盖对应剧集信息。详情页的 TMDB 相关推荐会优先展示已入库、评分较高、题材更接近且还没看完的内容，保险库锁定时不会把私密条目标成“在库”。音乐可选择 MusicBrainz 或 iTunes Search 作为索引源，并可在设置页或 macOS 菜单栏“媒体库 > 获取音乐封面和歌词”一键自动获取封面和歌词。设置页还提供“音乐元数据获取”工作台：点击“打开控制台”后，可按未匹配、全部音乐、缺少封面或收藏范围批量刮削音乐标签；结果会先显示在弹出层里，展开单条歌曲可修改标题、艺术家、专辑、曲序、年份、歌词和封面路径。默认只更新 MediaLIB 内部索引，不会改动原音乐文件；只有主动打开“写入文件”后，才会把所选结果写回本地音频文件标签。远程歌曲、不支持格式或不可写文件会显示失败原因，原文件不会被移动。
20. 媒体库上方筛选条已简化为小胶囊玻璃按钮，不再显示“范围/筛选/排序”等说明文字；视频分类和音乐歌曲/专辑/艺术家/歌单等页面切换会按当前栏目刷新，不会出现标题已变但列表仍停留在上一个分类的情况。页面切换尽量直接替换内容，减少整页动画造成的加载感；音乐播放器收起后侧栏会在底栏出现后恢复，避免和播放器转场同帧卡顿。除音乐展开播放器外，各页面使用统一暖白珍珠液态玻璃材质，主界面有从屏幕外左上方、靠近用户的位置倾斜射入的宽范围浅米白/香槟环境光，卡片、按钮、搜索框、菜单、弹窗、搜索结果和胶囊控件都会在左上受光面出现同方向偏白染色、高光和边缘轮廓光，右下只保留轻暗边。强调按钮和解锁按钮使用更深的蓝色玻璃层，避免浅色背景下文字发白看不清；普通搜索框和扫描按钮不再大面积泛蓝。鼠标经过交互卡片时会成为低饱和局部光源，光效范围更明显但仍保持局部，按钮、搜索框和菜单只在靠近鼠标的一侧边缘产生光线勾勒；长列表和静态卡片不会持续追踪鼠标，以保持滚动流畅。使用鼠标滚轮或触控板快速滑动时，MediaLIB 会临时暂停行 hover 放大、封面检视倾斜和指针液态光，停止滑动后恢复完整效果。音乐歌曲列表当前行参考媒体源 hover 反馈，使用固定高度暖白玻璃选中层、左侧光带、封面微缩放、白色高光和描边，右侧圆角会内收显示完整，不再大幅上浮或改变命中区域；歌曲列表、歌单歌曲明细、音乐专辑页和艺术家页都使用系统虚拟化列表，只挂载当前可见行和少量缓冲。专辑页仍保持网格卡片视觉，但由虚拟化列表按行承载，长时间滚动不会持续累积卡片视图。设置页使用惰性堆叠，视频海报墙使用分块瀑布流列表。本地海报会按显示尺寸载入缓存，减少大封面原图占用内存。如果系统开启“减少动态效果”，封面检视倾斜、hover 放大和连续鼠标光效会自动降级。如界面留白或对齐异常，优先反馈具体页面截图。
21. 2026-05-27 后，普通页面光源更接近低饱和象牙白/珍珠冷白，卡片和按钮通过白色填充、冷灰边缘和局部鼠标光源拉开层级；左侧栏更透明，更依赖系统模糊材质透出桌面背景。视频海报、剧集封面和音乐专辑封面的检视效果仍保留，但不会再放大外层命中区域，因此鼠标悬停时更不容易出现海报位移。音乐歌曲、专辑、艺术家和视频长列表都按可见区域惰性或虚拟化创建，不会一次渲染全部项目；切换视频/音乐分类、筛选或排序时，如果当前页面快照尚未准备好，会先显示轻量骨架占位，不会短暂显示上一页内容，也不会直接闪成“暂无”。底部音乐栏存在时，下层媒体库页面会自动进入 balanced 玻璃模式，保留白色厚玻璃、局部鼠标光、按钮边缘光和选中状态，但降低强度、提高位移阈值并减轻封面倾斜；迷你播放器进度条也改为轻量绘制控件，所以播放音乐时滚动设置页和长列表会更稳。hover 时只点亮当前行的固定高度绘制层、左侧光带和封面缩放，列表很长时响应会更紧。用滚轮或触控板浏览歌词也会临时降低远离歌词的 blur，只在自动跟随播放时恢复完整玻璃模糊；音乐页面之间切换会优先复用快照或显示轻量骨架，减少空白和加载感。展开/收起音乐播放器时侧栏会按统一非线性曲线分阶段收起/恢复，缩放窗口时歌词卡片也会限制在窗口内，避免把主窗口错误拉长。海报、歌词存在性和专辑取色缓存都设置了更明确的上限，音乐展开页节奏光效也调成更宽、更均匀的低对比柔光，减少色彩断层和长时间观看干扰。
22. 2026-05-30 后，所有带搜索和按钮的大标题栏都固定为标题左侧、搜索/扫描/清除记录等操作区右侧，操作区下边界与标题栏下边界对齐；搜索框和扫描/清除记录按钮共用同一套更厚实的暖白系统玻璃背景，更接近系统搜索控件。进入电视剧、动漫等视频页面并切换“全部”“正在观看”“已观看”“想看”“喜欢”等子页面时，标题栏、搜索框、扫描按钮和筛选条不会因列表宽度、滚动条或按钮数量变化而左右跳动。海报墙改为瀑布流，竖版海报和横版视频帧可自然错落但保持上下间距；海报标签会优先同排显示，空间不足时自动拆成两行。海报墙和专辑页的长标题不会换行或撑宽卡片，超出部分会在固定宽度内渐隐，鼠标悬停时首尾衔接循环滚动显示完整标题。专辑页封面使用足够清晰的缩略图缓存，滚动时应减少默认图与真实封面之间的闪烁。音乐展开页保留红黄绿按钮并隐藏侧栏展开按钮，展开/收起时外层只做透明度转场并延后切换 chrome，减少顶端露白和窗口撑高；界面按“多色专辑取色底板、整屏厚玻璃、封面/控制栏/歌词卡片、按钮”四层显示，控制栏首尾对齐且进度条自适应，封面附近的柔和光源会随播放节奏轻微位移和变色，但不会大范围染满整屏；歌词面板左上角不再有固定圆形光斑。
23. 2026-06-08 后，主窗口最小内容尺寸固定为 1088x720，播放器窗口也会按控制条安全尺寸和视频比例限制最小宽高。视频列表把“评分”和“评级”拆成两个排序项：评分始终表示 TMDB/Emby/Jellyfin/Plex 等资料源分数，评级表示用户点选的五角星；资料源评分只会在用户评级为空时折算出初始评级，之后以用户评级为准。EMBY / Jellyfin / Plex 服务端媒体库页面标题会使用左侧栏对应名称。音乐歌曲、专辑和艺术家页新增“最多播放/按播放次数”排序，专辑页的名称排序显示为“专辑名”，播放歌曲会累计次数，专辑和艺术家会按包含歌曲的播放次数汇总；处在最多播放页面时，右键歌曲、专辑或艺术家可重置对应播放次数，页头扫描按钮右侧也可一键重置所有歌曲播放次数。艺术家页只保留“全部”筛选，排序只保留按名称、按作品数量、按播放次数。点击播放歌曲不会让歌曲列表跳回顶部；从详情返回海报墙、首页看板或健康中心时会静默恢复到原位置，减少列表可感知跳动；点击歌曲列表吸顶表头会回到真正第一行；队列滚动位置只在本次打开软件期间记住；剧集列表鼠标按下就会更新选中态，双击仍直接播放。音乐展开页封面柔光使用预生成纹理和连续羽化平滑扩散，复制层不再改动 RGB 或叠加主色阴影，不会出现明显光圈断层或邻近颜色偏移。
24. 2026-05-31 之后的底部音乐栏收起更像封面横向移动：当底栏与列表或设置页同时存在时，只要开始滚轮或触控板滚动，不论方向，完整底栏会收起到右下角封面；封面会变暗并显示当前音频实时能量频谱，暂停时频谱静止，封面边缘会用专辑取色环形进度表示播放位置，点击封面会向左展开回完整底栏。海报墙顶部渐隐和轻模糊遮罩已移动到顶部过渡位置，海报滑出顶部时不会硬切或错位压住首行；音乐歌曲列表右下角恢复返回顶部按钮，音乐底栏存在时会自动上移避让。
25. 歌词同步算法可在“设置 > 音乐播放 > 歌词同步”中切换为快速估算、语速校准、音频校正或精确优先；所有选项都不会显示“甲/乙/丙”这类分档标签。普通逐句 LRC 会先预计算 estimated 逐字时间轴，避免当前词快结束时高亮仍停在开头；选择音频校正或精确优先时，本地音频会在后台生成 aligned 结果并缓存，生成前继续使用备用估算，完成后自动切换。拖长的字或带时长的逐字片段会降低上升速度，但歌词颜色、缩放和玻璃效果保持原样。
26. 歌词卡片右下角来源标记现在只显示“原词逐字 / 音频对齐 / 估算同步”等文字，不再显示前缀图标或“甲乙丙”样式。拖动音乐进度条后，歌词会直接使用本次拖动的目标时间查歌词时间轴：目标时间落入哪一句的开始时间和下一句开始时间之间，就把哪一句居中高亮；播放器会在 seek 落地前暂时屏蔽旧播放时间回写，避免歌词被拉回旧行。如果播放器真实落点随后确认到其他行，才会再校正到真实行。向前跳、向后跳或系统实际落点偏到前后几行时，会先把歌词视口直接居中到目标/真实行，之后再恢复正常播放时的平滑滚动。歌词卡片中心有柔和专辑色舞台光，白色只保留薄镜面高光；光只在卡片内且不盖到文字上。中心附近玻璃更透，上下边缘会更自然地模糊远离当前行的歌词。队列拖动排序时不会再反复跳回旧滚动锚点，长队列滑动和排序会更稳。底部音乐栏收起/展开时封面会在同一层向右/向左滑动，收起态进度环使用封面取色的邻近深浅色；展开页暂停时封面周边光效会连续收敛，封面轻微后退，环境柔光可略微照到歌词卡片和控制栏，继续播放时再平滑恢复封面和光晕。
