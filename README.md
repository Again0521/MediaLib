# MediaLIB

MediaLIB 是一款 macOS 原生家庭影音库。它可以把本地硬盘、移动硬盘、NAS、SMB/FTP 网络目录，以及 Emby、Jellyfin、Plex 里的电影、剧集、动漫、纪录片、综艺和音乐整理到一起，用一个干净的界面浏览、搜索、收藏、继续观看和直接播放。

它的目标不是替你接管文件夹，而是帮你管理“索引”。你原来的媒体文件会继续留在原来的位置；重分类、收藏、想看、播放记录、歌单、离线缓存、封面和元数据修正都保存在 MediaLIB 自己的本机数据里。

文档状态：2026-06-10

## 适合谁

- 影音文件分散在 Mac、移动硬盘和 NAS 上，平时靠 Finder 一层层翻。
- 电影、电视剧、动漫和音乐都想放在一个 App 里浏览。
- 家里有 Emby、Jellyfin 或 Plex，但也有大量本地文件。
- 希望有一个漂亮、安静、偏 macOS 原生风格的媒体库。
- 需要把私密目录锁起来，并且锁定时不显示路径、文件名和播放痕迹。

## 你可以用它做什么

### 接入不同来源

在“媒体源”页面点击“添加媒体源…”，按三步向导接入内容：

1. 选择来源：本地目录、网络设备、Emby、Jellyfin 或 Plex。
2. 完成连接：选择文件夹、填写网络地址，或登录媒体服务器。
3. 确认设置：选择分类、是否参与元数据拉取、是否参与片库健康检查，以及远程播放痕迹同步方式。

本地目录和网络挂载目录可以归类为电影、电视剧、动漫、纪录片、综艺、音乐、其他或保险库；不确定时也可以使用“自动识别”。网络设备会通过 macOS 挂载后再扫描真实目录，适合 SMB、FTP、FTPS 以及已经挂载到 `/Volumes` 下的 NAS。

Emby、Jellyfin 和 Plex 会显示为独立目录，不会混进本地“视频”或“音乐”分类。远程媒体库可以选择同步哪些服务器库，也可以选择播放记录和已观看状态的同步方式。

### 整理视频库

MediaLIB 会递归扫描目录，识别常见视频文件，并尽量从文件名和文件夹结构里判断它们是什么内容。

- 电影、电视剧、动漫、纪录片、综艺和其他分类分开展示。
- 剧集会按系列聚合，按季和集数排序。
- 支持 `S01E01`、`第01季 第02集`、`EP01` 等常见命名方式。
- 本地 `.nfo`、`poster`、`cover`、`fanart`、`backdrop` 等图片会优先使用。
- 没有封面时，可以截取视频帧、生成默认封面，或保持占位。
- MKV 等部分格式无法用系统方式截帧时，会尝试使用 App 内的 ffmpeg 兜底。

视频条目支持：

- 喜欢、想看、已观看、未观看、正在观看。
- 1 到 5 星个人评级。
- 按标题、年份、评分、评级、最近添加、播放次数等方式浏览。
- 清除单条或当前页面播放记录。
- 手动集合：适合自己整理专题片单。
- 智能集合：按类型、观看状态、年份、评分、评级、题材、来源等规则自动筛选。
- 集合发布到首页：把常看的片单放进首页总览。

“喜欢”和“想看”是两个独立状态：喜欢表示偏好，想看表示未来计划。手动标记已观看时，MediaLIB 会把它从想看中移出。

### 浏览音乐库

音乐是和视频并列的一套媒体库，不是视频库里的附属分类。扫描时会读取音频标签，尽量识别标题、艺术家、专辑、曲目号、年份、时长、内嵌封面和内嵌歌词。

你可以按这些入口浏览：

- 歌曲
- 专辑
- 艺术家
- 歌单
- 最近播放

收藏歌曲会出现在“音乐 > 歌单”里的置顶收藏歌单。你也可以创建自己的歌单，或把当前播放队列保存成歌单。歌单只记录 MediaLIB 内部索引和顺序，不会移动、复制或重命名音乐文件。

音乐库还支持：

- 歌曲右键加入队列、下一首播放、添加到歌单。
- 播放专辑、艺术家或歌单时，用这个分组替换当前队列。
- 队列清空、移除、点击播放和拖动排序。
- 随机播放、顺序播放、队列循环、单曲循环。
- 按播放次数浏览歌曲、专辑和艺术家。
- 读取 ReplayGain / R128 标签做响度均衡。
- 可配置跨曲过渡。
- Last.fm 听歌打卡。

歌词会优先使用音频内嵌歌词或同名 `.lrc` / `.txt` 文件。带时间戳的 LRC 可以自动滚动；增强 LRC 支持逐字或分词高亮。没有逐字时间戳时，MediaLIB 会根据歌词节奏估算行内进度，让歌词跟随播放更自然。

### 内置播放器

视频和音乐都可以用内置播放器，也可以在设置中改用系统播放器。

视频播放器支持：

- 常见视频容器和 MKV 等格式。
- 内嵌字幕、音轨切换。
- 同目录外挂字幕选择。
- 倍速、音量、全屏、窗口置顶。
- 进度条悬浮帧预览、剩余时间显示和恢复播放自动回退。
- 截图可选择带字幕、纯视频或窗口所见，并保存到用户图片目录下的 `MediaLIB 截图` 文件夹。
- 硬件解码策略和去色带设置。
- 章节、手动书签、A-B 循环、单片循环、片头/片尾标记和跳过片头/片尾。
- 远程视频清晰度选择。
- NAS 或挂载网络视频的播放端降采样。
- AirPlay 入口。
- 常用键盘快捷键、双击全屏、鼠标滚轮音量和触控板手势设置。

视频默认使用内置播放器还是系统播放器，只在“设置 > 视频播放与缓存”的启动方式里调整；播放器窗口齿轮菜单只管理当前视频动作和内置播放器行为。硬件解码、去色带、截图内容等参考 IINA、Infuse 和 mpv 常见能力补齐的选项，会放在内置播放器完整设置中，不会和默认播放器入口混在一起。

音乐播放器有底部迷你播放器和展开页两种状态。双击歌曲后会先出现底部迷你播放器，点击曲目信息后再展开整页播放界面。展开页会从专辑封面取色，显示封面、控制栏、队列和歌词卡片。

### 远程媒体服务器

MediaLIB 可以接入 Emby、Jellyfin 和 Plex。它会把远程服务器作为独立目录展示，并按服务器媒体库自适应生成入口。

远程来源支持的能力包括：

- 同步电影、剧集、单集和音乐条目。
- 使用远程海报和元数据。
- 保留服务器库分类。
- 播放远程流媒体。
- Emby / Jellyfin 可按设置同步播放进度、已观看和收藏。
- Plex 支持播放进度和已观看写回。
- 远程视频可以缓存到本地离线观看。

远程视频详情页不会显示带 token 的真实播放地址。服务器凭据保存在本机 Application Support 下的 MediaLib 数据目录中。

### 离线缓存

远程视频可以缓存到本地。缓存适合临时离线看剧，也适合网络不稳定时提前准备。

支持：

- 选择可用清晰度。
- 暂停、继续、取消缓存任务。
- 同步旁路字幕。
- 删除单集或整系列缓存。
- 系列自动缓存订阅，例如下一集、未看 3 集、未看 5 集、当前整季或全系列。
- 设置缓存到期和网络策略。
- 在首页“离线”标签查看已有缓存。
- 设置缓存容量上限，并通过一键清理回收过旧或已看的缓存。

缓存只管理 MediaLIB 自己生成的缓存文件，不会删除媒体源里的原始文件。

### 首页、搜索和推荐

首页可以显示总览、下一集、继续观看、离线、最近添加、各分类、喜欢、未观看和保险库入口。没有内容的标签不会强行显示。

总览会把继续观看、下一集、推荐和已发布集合放在一起。推荐会参考你看过、喜欢、想看和标星的题材，也会结合资料源评分，优先展示更可能想看的内容。

首页搜索框可以跨视频和音乐搜索。左侧栏里也可以直接进入视频分类、音乐分类、远程服务器、片库健康、任务中心、媒体源和设置。

### 片库健康和任务中心

“片库健康”用于检查媒体库是否有这些问题：

- 媒体源离线。
- 本地文件路径失效。
- 远程播放路径缺失。
- 疑似重复条目。
- 缺少封面、年份、简介或音乐标签。

健康中心里的清理动作都需要确认，并且只会移除 MediaLIB 内部索引，不会删除、移动或修改媒体文件。离线来源会受到保护，保险库锁定时不会暴露私密路径和条目名。

“任务中心”会记录扫描、增量扫描、远程同步、封面预热、元数据补充、视频缓存、章节图和一键清理等后台任务。失败任务在可以重建目标时会显示重试按钮。

### 保险库

保险库适合放私密内容。第一次进入时可以设置 4 到 8 位数字密码，之后可用 Touch ID 或密码解锁。

锁定时：

- 不显示保险库内容。
- 不显示保险库媒体源路径。
- 不显示扫描中的私密文件名。
- 不在首页、正在观看、已观看、喜欢或想看中泄露保险库条目。
- 不显示改密入口。

解锁后，保险库内容可以正常浏览、播放、收藏、标记和清除播放记录。

### 元数据和标签

影视信息可以通过 TMDB 匹配，音乐信息可以在设置中选择 MusicBrainz、iTunes Search、网易云音乐、QQ 音乐、Last.fm 或 Deezer 等来源。

MediaLIB 支持：

- 在详情页搜索并应用影视或音乐信息。
- 一键补充缺失封面、简介、年份和音乐标签。
- 批量打开“音乐元数据获取”工作台。
- 在写入前预览和编辑标题、艺术家、专辑、曲序、年份、歌词和封面。
- 默认只更新 MediaLIB 内部索引。
- 只有你主动打开“写入文件”时，才尝试写回本地音频文件标签。

元数据覆盖会保留历史记录，可以在详情页或设置中撤销最近一次覆盖。撤销只影响 MediaLIB 内部索引，不会移动媒体文件。

## 数据安全

MediaLIB 的基本原则是：不擅自改动你的媒体文件。

- 扫描只建立索引。
- 重分类只修改 MediaLIB 内部分类。
- 喜欢、想看、评级、播放记录和歌单都保存在本机索引里。
- 清理失效索引不会删除原始文件。
- 删除离线缓存只删除 MediaLIB 自己生成的缓存副本。
- 保险库锁定时会隐藏路径、文件名和内容。
- 数据库备份和恢复只处理 MediaLIB 内部数据，不复制或替换媒体源里的文件。

设置里的“数据、存储与诊断”可以创建数据库备份、打开备份目录、从备份恢复，也可以执行一键清理。恢复前会先保存当前数据库，避免直接覆盖。

## 快速开始

1. 打开 `MediaLIB.app`。
2. 进入“媒体源”，点击“添加媒体源…”。
3. 选择本地目录、网络设备、Emby、Jellyfin 或 Plex。
4. 本地或网络目录选择分类；不确定时使用“自动识别”。
5. 点击“扫描全部”，或扫描单个媒体源。
6. 扫描完成后，在左侧栏进入“首页”“视频”或“音乐”浏览。
7. 如果需要联网补充信息，在“设置 > 元数据与匹配”里配置 TMDB 或音乐数据源。

首次使用时建议先添加一个较小的目录试扫，确认分类和封面效果符合预期后，再添加完整媒体库。

## 安装

如果你拿到的是 `MediaLib.dmg`：

1. 打开 DMG。
2. 将 `MediaLIB.app` 拖到“应用程序”。
3. 首次打开未签名版本时，macOS 可能会拦截。右键 `MediaLIB.app`，选择“打开”，再确认一次即可。

运行要求：

- macOS 13 Ventura 或更高版本。
- 本地扫描和播放不需要账号。
- TMDB、音乐元数据、远程服务器、字幕下载、Trakt 和 Last.fm 等功能需要网络和对应服务的账号或 API Key。

## 从源码构建

普通使用不需要看这一节。如果你想自己构建：

```bash
brew install mpv ffmpeg
env DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer swift build
env DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer swift run MediaLibChecks
scripts/package_dmg.sh
```

构建产物位于：

- `dist/MediaLIB.app`
- `dist/MediaLib.dmg`

项目使用 Swift Package Manager，主要 target 包括：

- `MediaLib`：macOS App。
- `MediaLibCore`：模型、扫描、数据库和服务。
- `MediaLibChecks`：轻量自检工具。
