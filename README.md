# MediaLIB

**一款 macOS 原生的家庭影音库**

把散落在本地硬盘、移动硬盘、NAS 和 Emby 服务器上的电影、剧集和音乐，整理成一个干净、好看、能直接播放的媒体库。

</div>

---

## 这是什么

MediaLIB 是给 Mac 用户做的一个本地优先的影音管理工具。

你大概率有这样的场景：电影存在移动硬盘里，动漫放在 NAS 上，音乐是多年攒下的一堆 FLAC 和 MP3，家里可能还跑着一台 Emby。平时想看一部片子，要么翻 Finder，要么开好几个 App。MediaLIB 想做的事很简单——把这些来源接进来，自动认出每个文件是什么、属于哪一季哪一集、是哪张专辑，再用一面海报墙把它们摆好，点一下就能播。

它完全用 SwiftUI + AppKit 写成，播放内核是内嵌的 libmpv，索引存在本地 SQLite 里。没有服务端，没有账号，数据都在你自己的电脑上。

## 几个它比较在意的点

**认得出你的文件。** 电影、剧集的文件名识别支持 `S01E01`、`第01季第02集`、`EP01` 等常见写法；电视剧会自动按系列归拢、按季按集排好，而不是把同一部剧拆成一堆零散标题。经典的 Emby/Jellyfin 目录结构会优先按剧名文件夹聚合。音乐扫描会老老实实读音频标签——标题、艺人、专辑、曲序、年份、内嵌封面和内嵌歌词都读，FLAC/MP3/M4A 里藏的标签也尽量挖出来，而不是把整个文件夹粗暴地当成一张专辑。

**封面尽量帮你补齐。** 本地有 `poster`/`cover`/`fanart`/`.nfo` 就优先用本地的；没有的话，可以连 TMDB 搜电影和剧集信息，连 MusicBrainz 或 iTunes 补音乐封面和歌词。实在没有图，可以截一帧视频画面或画一张默认封面顶上。所有联网拿到的图都会缓存到本地再用。

**界面是认真做过的。** 整体是一套偏暖白的「液态玻璃」风格，光从左上角斜进来，卡片、按钮、搜索框都有统一的材质和层级。海报墙会按竖版/横版的真实比例自适应排布，长标题在卡片里悬停时会平滑地循环滚动。音乐展开播放页会从专辑封面取色，做成沉浸式的背景光和歌词卡片，歌词支持逐字高亮，效果接近 Apple Music。

**播放器是内置的。** 视频用内嵌 libmpv 渲染，支持 MKV 等常见容器、字幕轨/音轨切换、同目录外挂字幕、倍速、进度条悬浮帧预览、全屏，以及 AirPlay。音乐用原生 AVFoundation 后端，底部有迷你播放器，点开能铺满整屏。系统媒体键和菜单栏也能控制播放。

**私密内容可以锁起来。** 「保险库」分类支持用 Touch ID 或数字密码上锁，锁定时连媒体源路径和文件名都会隐藏，也不会出现在首页、最近播放或收藏里。

## 主要功能一览

- **多种媒体源**：本地目录、移动硬盘、`/Volumes` 下挂载的 NAS、SMB/FTP 网络设备，以及 Emby 账号登录同步
- **自动分类**：添加时引导选择类型（电影 / 电视剧 / 动漫 / 纪录片 / 综艺 / 音乐 / 其他 / 保险库），之后还能改
- **视频库**：海报墙、系列聚合、剧集列表、收藏、评分、已看/未看标记、正在观看
- **音乐库**：歌曲、专辑、艺术家、歌单、最近播放；歌单本地持久保存，不会动你的音乐文件
- **内置播放器**：libmpv 视频 + AVFoundation 音乐，字幕/音轨/倍速/清晰度/AirPlay 一应俱全
- **元数据刮削**：TMDB（影视）、MusicBrainz / iTunes（音乐），可批量获取音乐封面与歌词
- **保险库**：Touch ID / 数字密码加锁的私密分区
- **Emby**：登录同步，按服务端分类显示，远程流播放并支持转码清晰度选择
- **自动扫描**：可设定间隔，自动发现新增内容

## 安装与构建

需要 macOS 13 及以上，以及通过 Homebrew 安装的 `libmpv` 和 `ffmpeg`（打包时会一并装进 App，分发后的机器无需再装）。

```bash
brew install mpv ffmpeg
scripts/package_dmg.sh
```

构建产物：

- `dist/MediaLIB.app`
- `dist/MediaLib.dmg`

由于是未签名版本，首次打开时如果被 Gatekeeper 拦下，右键 App 选择「打开」即可。

也可以直接用 Swift Package Manager 跑起来：

```bash
swift build      # 编译
swift run MediaLib   # 运行
```

## 技术栈

| 层 | 用的东西 |
|---|---|
| 界面 | SwiftUI，必要处用 AppKit |
| 视频播放 | 内嵌 libmpv（通过 render API 绘制到自有 OpenGL 视图） |
| 音乐播放 | AVFoundation |
| 数据 | 本地 SQLite |
| 元数据 | TMDB / MusicBrainz / iTunes / OpenSubtitles |

代码大致分三个 target：`MediaLibCore`（模型、数据库、扫描、服务）、`MediaLib`（界面与播放）、`MediaLibChecks`（一个轻量自检命令行工具）。
